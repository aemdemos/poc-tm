6. Efficiency Optimizations

Key Insight: 8 templates cover all 789 pages. Investing in robust template infrastructure upfront dramatically reduces per-page migration effort.
  1. Template Reuse: The blog/article template alone covers ~40% of all pages (~315 pages). Blog, news, podcasts, and legislative highlights all share the same basic article structure, enabling ~330 pages to use the same import script.
  2. Block Variant System: Rather than creating new blocks for each visual pattern, use CSS-only variants of core blocks (cards, hero, carousel). This reduces JavaScript development and maintenance burden.
  3. Skip Taxonomy Archives: The 92 category + tag archive pages can be replaced with Edge Delivery Services’ query-index mechanism + search block. This eliminates them from the migration scope entirely.
  4. Batch by Similarity: Group content types that share templates (blog + news + podcasts + legislative highlights) into single bulk import runs with minor conditional logic.
  5. Effective Page Count: After removing taxonomy archives (92) and grouping by template, the actual migration work is: 8 template setups + ~690 content pages through bulk import.
  6. Import Infrastructure Investment: Phase 2 builds block parsers and page transformers once per template. Phase 3 reuses these across hundreds of pages with minimal manual intervention.

8. Risk Considerations

Risk Severity Mitigation
Lottie animations cannot be directly replicated MEDIUMYellow Replace with static images or CSS animations. Lottie library can be loaded as delayed script if required.
HubSpot form integration complexity LOWGreen HubSpot forms can be embedded via iframe. Embed block supports this pattern.
AJAX-loaded content in audience accordions MEDIUMYellow Content will need to be pre-loaded during scraping or fetched from source during import.
Mega menu complexity MEDIUMYellow EDS navigation supports multi-level menus. May require custom header decoration for mega-menu dropdowns.
Animated number counters LOWGreen Implement with IntersectionObserver + requestAnimationFrame in stats block JS.
Cookie consent management LOWGreen Ketch or equivalent can be loaded as delayed script in EDS.
Search/filter on archive pages MEDIUMYellow EDS search block + query-index provides base. Custom filtering (by type + topic) may need additional JS.

Begin with Phase 1: Design system extraction to establish the visual foundation, followed by header/footer since they appear on every page. Then proceed to migrate the first blog post (Phase 2.1) to establish the highest-volume template and unlock Phase 3 bulk import for ~315 pages.
